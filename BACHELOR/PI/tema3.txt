
Pregunta 2. Identifica 3 diferencias entre el encaminamiento intra-domain y el encaminamiento inter-domain.

1. Alcance y Aplicación
Encaminamiento Intra-Dominio:
Alcance: Se aplica dentro de un solo sistema autónomo (AS). Es utilizado para dirigir el tráfico dentro de una red específica, como la red interna de una empresa o proveedor de servicios de Internet (ISP).

Encaminamiento Inter-Dominio:
Alcance: Se aplica entre diferentes sistemas autónomos. Es utilizado para dirigir el tráfico entre múltiples redes independientes, como entre diferentes ISPs o entre grandes redes empresariales.

2. Políticas y Métricas
Encaminamiento Intra-Dominio:
Políticas: Se centra en la optimización del rendimiento dentro de una red, utilizando métricas como el ancho de banda, el costo del enlace, la latencia, y la confiabilidad para determinar las rutas óptimas.

Encaminamiento Inter-Dominio:
Políticas: Las decisiones de enrutamiento están basadas más en políticas de enrutamiento que en métricas técnicas. Estas políticas pueden incluir acuerdos comerciales, preferencias de los proveedores, y consideraciones de seguridad.

3. Convergencia y Complejidad
Encaminamiento Intra-Dominio:
Convergencia: Los protocolos intra-dominio tienden a tener tiempos de convergencia más rápidos debido a su diseño para funcionar dentro de un entorno más controlado y de menor escala. Los routers tienen una visión completa y detallada de la topología de la red interna.

Encaminamiento Inter-Dominio:
Convergencia: Puede tener tiempos de convergencia más lentos debido a la escala y la naturaleza descentralizada de Internet, donde múltiples AS independientes deben sincronizarse. Además, las políticas de enrutamiento pueden complicar el proceso de convergencia.

El Patricia tree Algorithm consiste en organizar la tabla de encaminamiento de forma jerárquica. De este modo tenemos en el primer nivel las IP's con menor bits de máscara y de estas IP cuelgan otras subredes IP pertenecientes a la red IP mayor, en forma de árbol, de esta forma no hay que hacer demasiados saltos para encontrar la red IP con mayor máscara de bits.

State. b) ¿Cómo se realiza el flooding en una red OSPF con un único área?, c) ¿Cómo se realiza el
flooding en una red OSPF multi-área?
El mecanismo de flooding consiste en que los routers de una red consigan tener una base de datos con una topología de la red. En los protocolos link State los routers van anunciando a los routers vecinos lo cambios que vayan recibiendo de la topología de la red, por ejemplo si se cae un enlace. EN OSPF en un único área una vez elegidos los DR's y BDR's de las redes del área, el DR es el encargado de actualizar la base de datos de los DROTHERS, estos Drothers avisan únicamente al DR cuando sucede un cambio en la topología entre él y sus vecinos. El BDR sirve de backup para el DR. En una red multi-área los routers internos de cada área solamente tienen una base de datos con la topología de su área, son los ABR's y ASBR's, quienes tienen más ed una base de datos, tantas como areas adjacentes.

Pregunta 21. Enuncia los tipos de LSA’s OSPF qué hay en una red multi-área y que funcionalidad
tienen dentro del esquema OSPF multiárea.

Type 1: Router LSA:
 generated by each router inside its area towards each internal router,
 describes each link state and cost for each internal router,
 Type 2: Network LSA:
 generated by each DR (in a BMA network), describes the set of routers
connected to that BMA network (only sent inside the area),
 Type 3: Summary LSA:
 generated by each ABR router, describe external routes (got from other
ABR’s),
 There is an entry per each subnetwork (summarized),
 Type 4: ASBR summary LSA:
 generated by each ABR router, describe routes towards ASBR’s (to get
out the AS),

Type 5: AS external LSA:
 generated by ASBR routers, describe external routes belonging to other AS (e.g.,
default network to get out the AS),
 used in standard areas that are not “Stub Areas” or “No So Stubby Areas”
 Type 7: NSSA (Not So Stubby Area) external LSA:
 generated by ASBR routers, describe external routes belonging to other AS (e.g.,
default network to get out the AS),
 only used by Not So Stubby areas (NSSA).
