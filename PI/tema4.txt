Pregunta 1. Explica que es una política de encaminamiento y como se implementa.
La política de encaminamiento se refiere a la función de buscar un camino entre todos los posibles en la red a partir de unos atributos que no son metricas técnicas.

Pregunta 2. Explica como escala la tabla de encaminamiento BGP en función de la cantidad de AS’s a
los que está conectado un AS.
Escala de forma ((#número de AS's) * (#número de rutas)), porque en la tabla de encaminamiento de BGP hay todas las posibles rutas para llegar a cada AS, de estas rutas después se escoge la mejor para la tabla de encaminamientos del router.

Pregunta 3. ¿Pará que sirve definir una dirección de loopback en un router? ¿Qué tipo de dirección
es?
Es una dirección vitual y sirve para evitar pérdidas de conectividad cuando se cae una interfaz física, de esta forma los routers que quieran enviar a esa dirección de loopback recalcularán la ruta para acceder desde otra interfaz física.

Pregunta 4. ¿Cómo resuelve BGP el problema de los blucles?
Con el AS-Path vector, consiste en un vector en el cual a medida que el paquete va pasando por AS's, los va poniendo en este vector. En caso de volver a pasar por un AS, el router verá que el AS se repite y descartará el paquete.

Pregunta 5. ¿Qué diferencia hay entre IBGP e EBGP?
IBGP se utiliza para coordinar políticas de encaminamien entre routers de un mismo AS y EBGP para routers entre AS. Entre routers IBGP no se anuncian rutas de encaminamiento aprendidas por IBGP, para evitar repetir el AS en el AS-Path vector.

Pregunta 15. Justifica porqué los routers BGP tienen que estar i-BGP totalmente mallados y
explica la diferencia entre el funcionamiento i-BGP y e-BGP respecto al anuncio de rutas.
Porque los routers que aprenden rutas por i-BGP no lo pueden enviar a otros routers con conexión I-BGP, por lo tanto se necesita una malla completa para que se pueda propagar la ruta anunciada a otros AS's. La diferencia está en que las rutas aprendidas por e-BGP pueden ser enviadas tanto por e-BGP como por i-BGP. En el anuncio de rutas, cuando se anuncia una ruta de a través de una conexión e-BGP el router que anuncia la ruta incluye su AS en el AS-path vector, en cambio cuando se anuncia a través de i-BGP no se incluye el AS.

El multi-homing es cuando un AS tiene más de una conexión a un ISP o más. Pregunta 18. Asume que tienes un ISP con 100 routers BGP. Indica cuantas sesiones I-BGP necesita
para funcionar correctamente. Indica que tecnicas hay para reducir el número de
sesiones I-BGP y explica brevemente el funcionamiento de una de ellas. Pon un
ejemplo de las técnicas que has explicado enseñando la reducción de sesiones BGP a
los 100 routers.

Número de sesiones: (N(N-1))/2 = (100*99)/2 = 4950 sesiones

Se pueden utilizar route reflectors, o bien confederaciones.

Los route reflectors consisten en dividir el conjunto de routers en clusters, en cada cluster un route reflector, entre route reflectors tiene que haber una malla completa de sesiones I-BGP, en cada cluster basta con que cada router BGP tenga una sesión IBGP con el route reflector. Si un route reflector aprende una ruta via otro route reflector, lo envía a sus clientes. Si un route reflector aprende una ruta a través de una conexión e-BGP o un cliente, este lo reenvía a todos sus vecinos.

Las confederaciones consisten en crear AS's privados dentro de un AS, de forma que en cada AS privado debe haber almenos un router con una conexión ei-BGP con un router de otra confederación. Dentro del AS privado tiene que haber una malla completa de sesiones i-BGP.
